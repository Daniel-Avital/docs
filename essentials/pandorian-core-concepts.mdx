## What is a Guideline?

A guideline is a software development rule or best practice written in plain English that Pandorian enforces across your codebase. Instead of configuring complex static analysis tools, you describe what you want in natural language.

**Example Guidelines:**

*"When initializing lists and dictionaries with known quantities, consider pre-sizing to reduce allocations and improve performance."*

*"All configuration values including URLs, timeouts, limits, and paths must be externalized to configuration files or environment-specific sources rather than embedded directly in source code."*

Guidelines capture your team's standards, architectural decisions, security requirements, and performance best practices in a centralized, enforceable format. They work across any programming language and automatically adapt to your codebase's specific context.

### Anatomy of a Guideline

Every guideline consists of:

**Title:** A clear, actionable name that describes the rule (e.g., "Pre-size Collections When Possible" or "Hardcoded Configuration Values Must Be Eliminated")

**Description:** Plain English explanation of what should or shouldn't happen in your code — including context, reasoning, and any exceptions.

**Tags:** Metadata that helps organize and prioritize guidelines:
- **Category** (Performance, Security, Configuration, Code Quality, etc.)
- **Enforcement** (Block or Monitor)
- **Language** (Python, Java, JavaScript, or language-agnostic)

## PR Scan vs. Full Codebase Scan

Pandorian offers two scan modes depending on what you need.

**PR Scan:** Analyzes only the changes introduced in a pull request. Runs automatically via GitHub App integration, giving developers fast feedback before merge without re-analyzing the entire codebase.

**Full Codebase Scan:** Analyzes an entire repository against all active guidelines. Use this for audits, onboarding new repos, or getting a complete picture of where your codebase stands against your standards.

---

## How Codebase Governance Enforcement Works

When a guideline is created or updated, Pandorian's AI analyzes its description to understand the pattern or anti-pattern to detect, the context that makes a violation legitimate vs. problematic, and any edge cases implied by the description.

Scans can be triggered on-demand, on pull requests, or via CI/CD pipeline. When a scan runs, Pandorian analyzes your code against all active guidelines, identifies violations with file path and line-level context, and generates remediation guidance for each finding.

Guidelines set to **Block** will fail CI/CD pipelines and prevent merge. Guidelines set to **Monitor** surface violations for visibility without interrupting development.

---

## Scan Results & Violations

After a scan completes, results are available in the Scans page. Each violation includes the file path and line number, an explanation of why the code violates the guideline, and the option to generate a context-aware fix instruction — a markdown file you can review yourself or feed directly to an AI coding tool like Copilot, Cursor, or Claude.
